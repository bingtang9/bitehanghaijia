# 用 RSSHub + OpenClaw 搭一个自动帮你刷推的系统

> Twitter 推文 + 深潮文章 → AI 筛选总结 → 龙虾定时推给你

## 起因

我在 Twitter 关注了一百多个博主。做投资嘛，各路分析师、KOL、媒体号都得跟着。

问题是根本刷不过来。

一天不看就攒几百条。打开一看，一半是碎碎念，三成是转发互动，真正有信息量的可能就十来条，但偏偏淹在里面找不到。算法推荐也不靠谱，经常让你看到三天前的推文，该看的反而刷不到。

另外深潮 TechFlow 上有不少深度文章值得读，但跟 Twitter 是两个平台，来回切也烦。

所以搞了这套东西：让机器每小时去抓一遍，AI 筛掉水的，把有价值的内容总结好推给我。我只看精华就行。

跑在 Mac mini 上，不花钱。


## 它干了什么

每小时从 Twitter 和深潮拉最新内容。AI 看完之后分两路推：投资相关的走一个龙虾，科技生活类的走另一个。水贴广告直接扔掉。

晚上十点再来一份全天总结，防止白天忙漏看了什么。

## 你需要

- 一台常开的机器，Mac mini 或 Linux 都行
- Docker
- OpenClaw
- 一个 Twitter 小号的 cookie


## 怎么搞的

### RSSHub 抓 Twitter

RSSHub 是个开源项目，能把各种网站转成 RSS。Docker 起一个容器，把 Twitter 小号的 cookie 塞进去就能用。

用小号是因为 cookie 约等于登录态，被检测到风险不大但也不是零。小号挂了无所谓。

### List 是关键

一开始想逐个抓 111 个博主的时间线。跑起来才发现 Twitter 限流很凶——每个请求要等二十多秒，批量跑直接全超时。

后来改成建一个私有 List，把所有人丢进去。RSSHub 有 List 路由，一个请求就能拿到大约 70 条最新推文。

批量往 List 里加人用的是 Twitter 的 GraphQL API，写了个脚本循环调用，每加一个人歇 1.5 秒。111 个人跑完零失败。

### 深潮没有文章 RSS

深潮的 rss.xml 只有快讯，就是那种一两句话的短消息。深度文章要另想办法。

它是 Next.js 做的，请求文章页的时候加一个 `RSC: 1` 的 header，服务端会返回结构化数据，从里面能提取文章列表和正文。

麻烦的是正文有三种不同格式——有的是直接 HTML，有的是 Unicode 转义的，有的只能从 meta description 里捞。脚本得三种都兼容。实际跑下来大概 90% 的文章能拿到正文。

### 抓取脚本

两个 Python 脚本，逻辑差不多：拉数据 → 跟本地记录比对去重 → 新内容存到待推送文件。

Twitter 那个多了一步：如果抓取失败（一般是 cookie 过期了），会写一个错误标记，定时任务检测到就通知我去换 cookie。

去重文件会自动清理，只留最近的记录，不会越长越大。

### 定时任务

一共 7 个：

| 干什么 | 频率 |
|--------|------|
| 抓取 | 每小时，静默执行不打扰 |
| 推送 | 白天 5 次（8/11/14/17/20 点） |
| 全天总结 | 晚 10 点 |

晚上 11 点到早 8 点只抓不推，攒到早上一块儿发。


## 分类怎么分的

投资龙虾收到的：行情分析、宏观经济、监管动态、交易策略、深潮金融类文章。

生活龙虾收到的：AI 和科技产品、社会观察、观点杂谈、深潮科技类文章。

内容质量不一样处理也不一样。深度分析给两三句详细摘要，普通推文一句话带过，水贴广告直接不发。

深潮文章单独一个板块，每篇附几句正文摘要，看完摘要就知道要不要点进去读全文。

## 踩过的坑

**逐个抓不行。** 一百多个人逐个请求，Twitter 限流到每个要等 20 秒，跑到一半全超时。改用 List 一把搞定。

**Python 调 Twitter API 返回 404。** 同样的请求 curl 没问题但 Python urllib 不行，查了半天发现是 urllib 不处理 cookie 和重定向。最后批量加 List 成员用的 bash + curl。

**深潮的 RSS 是假的。** 说是有 rss.xml，打开一看全是快讯。深度文章得走 Next.js 的 RSC 接口，正文格式还不统一。

**Cookie 过期没人知道。** RSSHub 抓取失败不报错，默默返回空数据。不在脚本里加检测的话，你以为系统在跑其实早停了。

**去重文件会膨胀。** 跑久了 seen 文件越来越大。加了自动清理，只保留最近几千条。


## 后面可以搞的

PANews、Odaily 这些有 RSS 的媒体可以用同样的套路接进来。RSSHub 支持多个 cookie 轮换，减少单个小号的压力。微信公众号目前没有靠谱的免费方案，先不碰。
