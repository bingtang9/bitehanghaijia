## Problem Statement

用户想构建一个 Polymarket 做市机器人，通过在体育赛事市场挂限价单获取流动性奖励。朋友已经在做，每天约 1% 回报，验证了可行性。用户需要自己的机器人，核心关注点是钱包安全和策略实现。

## Demand Evidence

- 朋友已经用自研机器人做市，日回报约 1%
- Polymarket 每月分发超过 $5M 流动性奖励（2026年4月数据）
- 多个开源做市机器人项目存在，说明有持续的开发者需求
- 官方提供完整的 CLOB API 和多语言 SDK 支持

## Status Quo

朋友的现有方案：
- 自研做市机器人
- 独立热钱包 + 助记词分片保存
- 策略细节未公开

行业现状：
- 所有开源方案把私钥存在 .env 文件，安全性差
- 官方 poly-market-maker 支持 AMM 和 Bands 策略，但不活跃维护
- 竞争加剧，纯奖励farming利润收窄

## Target User & Narrowest Wedge

目标用户：用户自己（先验证再产品化）

最小可行版本：
- 一个能在 2-3 个体育市场自动挂双边限价单的 Python 机器人
- 私钥加密存储，不用 .env 明文
- 能追踪 PnL 和奖励收入
- 有基本风控（单市场敞口上限）

## Constraints

- 用户对做市策略细节不熟悉，需要从简单策略起步
- 需要启动资金（建议 $1,000-5,000 起步）
- Polymarket 流动性奖励可能随时间变化（补贴性质）
- 体育赛事有明确结算时间，库存风险在赛事结束时集中爆发

## Premises

1. Polymarket 流动性奖励是真实且持续的收入来源（目前 $5M+/月）— 已确认，但需持续监控奖励池变化
2. 做市的核心风险不是技术，而是策略 — 赛事结果可能导致持仓偏向一边，全额亏损 — 已确认
3. 每天 1% 回报不是无风险的，会有亏损的日子 — 已确认
4. 需要先用小资金跑通策略，再放大 — 已确认

## Cross-Model Perspective

独立 AI 分析要点：
- 最强版本：私钥安全的做市机器人，长期产品化为"安全基础设施即服务"
- 关键发现：朋友已在赚钱 = 需求验证已完成，不需要教育市场，只需要执行

## Recommended Approach

方案 A — 最小可行版。原因：用户策略还不清楚，需要快速跑起来用真实数据验证。自己从零写比 fork 更了解每一行代码，方便后续迭代。加密 keystore 解决安全核心需求。

### 技术架构

```
polymarket-mm-bot/
├── main.py              # 主循环：获取市场数据 → 计算报价 → 挂单
├── strategy.py          # 做市策略（对称挂单 + spread 计算）
├── risk_manager.py      # 风控（敞口上限、止损、赛事结束平仓、MATIC余额检查）
├── wallet.py            # 加密 keystore 管理（eth_account keystore v3）
├── pnl_tracker.py       # PnL 追踪（SQLite）
├── config.yaml          # 配置文件（含市场列表硬编码）
└── requirements.txt
```

MVP 阶段不做：market_selector.py（市场在 config.yaml 硬编码）、dashboard.py（用日志查看）。第二阶段再加。

### 核心策略逻辑

1. 通过 CLOB API 获取市场 orderbook，midpoint = (best_bid + best_ask) / 2
2. 通过 Markets API 获取 min_incentive_size 和 max_incentive_spread 参数
3. 对称挂单：bid = midpoint * (1 - spread/2)，ask = midpoint * (1 + spread/2)
4. spread 初始值建议 max_incentive_spread 的 50%，后续根据数据调优
5. 订单大小 = max(min_incentive_size, 配置的默认单量)，从 API 动态获取 min_incentive_size
6. 每 60 秒同步一次（避免触发 API 速率限制）：当报价偏离当前 midpoint > 1% 时撤旧单重挂
7. 异常处理：API 连续失败 3 次暂停该市场 5 分钟，记录错误日志
8. 单市场库存偏移控制：YES/NO 持仓差超过 $200 停止该市场挂单，单市场总敞口上限 $500
9. 赛事结束前平仓：赛事结束前 1 小时自动撤销该市场所有挂单，不再接受新仓位，尝试市价清理剩余持仓

### 认证流程

1. 通过 py-clob-client 使用私钥生成 L1 认证（EIP-712 签名）
2. 获取 API Key（L2 认证）用于后续 API 调用
3. 首次使用前需通过 Polymarket UI 完成至少一笔交易（设置链上权限）

### 安全方案

- 使用 eth_account 库的 encrypt/decrypt（Ethereum keystore v3 标准格式）
- 启动时输入密码解密
- 运行时私钥只在内存中
- 不写日志、不传网络
- 监控 MATIC 余额，低于阈值（如 0.1 MATIC）暂停交易并告警

以下问题留待实现阶段解决：
1. py-clob-client SDK 维护状态需确认，如已停更则需手写 EIP-712 签名（复杂度显著提升）

5. USDC 充值到 Polygon 链上的具体操作流程需要在实现时补充文档
